Message Chain

Message Chain smell code is when a client repeatedly calls methods on the return value of other methods, forming a long chain of method calls.

When does it happen?

When the client repeatedly calls methods or accesses properties of one object, which in turn calls methods or accesses properties of another object, and so on, forming a chain of dependencies, resulting in a chain like obj.a().b().c().

Treatment

Consider utilizing Hide Delegate to remove message chains. Additionally, think about the purpose of the end object and consider employing Extract Method to centralize functionality, then utilize Move Method to relocate it to the beginning of the chain.

Example

Consider DistanceTest.java, there is message chaining when accessing latitude and longitude.

DistanceTest.java


            ...
            driver.getCurrentPosition().getLatitude();
            ...

Treatment:

Hide Delegate has been implemented. Notice in the BojekDriver and Destination classes, functions latitude() and longitude() have been added, actually delegating the chaining performed in the previous package. This is done so that the client class doesn't feel the chaining, hidden within here.

DistanceTest.java


            public double latitude() {
              return this.getCurrentPosition().getLatitude();
            }
            
            public double longitude() {
              return this.getCurrentPosition().getLongitude();
            }
